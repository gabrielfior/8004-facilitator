Phase 0: Agent Registration & Discovery

How agents register and how clients discover them

Agent registration (on-chain):

  • Agent generates a fresh EOA and calls IdentityRegistry.register() on-chain
  • This mints an ERC-721 NFT with a unique agentId (e.g. 33061)
  • The agent's EOA is the NFT owner. The agent can set tokenURI(agentId) to a JSON registration file listing their services

Agent discovery (how clients find agents):

  • Shared via directory, marketplace, or known listing
  • Client queries IdentityRegistry.tokenURI(agentId) to fetch registration file with service endpoints (signing API, resource server URL, etc.)

Phase 1: Payment Flow

Standard x402 — same as any vanilla x402 payment ERC-8004 Δ — payee is Agent EOA, not Server wallet
1
Client → Resource Server x402
GET /weather with no payment header. Server returns 402 + X-Payment-Requirements: pay USDC $0.01.
2
Client → Facilitator x402
Client signs EIP-3009 TransferWithAuthorization. Sends POST /verify. Facilitator validates signature + simulates.
3
Client → Resource Server x402
Retries GET /weather with PAYMENT-SIGNATURE header. Middleware intercepts.
4
Server → Facilitator x402
Server calls POST /settle. Facilitator executes transferWithAuthorization.
4a
Settlement beneficiary ERC-8004
In vanilla x402, USDC goes to the Server's wallet. In our flow, pay_to is the Agent's EOA. The Server never touches the payment — it's purely a frontend for the Agent's service.
5
Server → Client x402
Weather data returned (200 OK). PAYMENT-RESPONSE header includes settlement tx hash.

Phase 2: Proof of Work & Reputation (entirely new)

All new — doesn't exist in vanilla x402
6
Client → Agent's signing endpoint new
Client sends POST /sign-proof with { settlementTxHash, requestBody, responseBody }.
Agent verifies the settlement tx on-chain (checks receipt, confirms USDC sent to Agent's EOA). If valid, signs keccak256(agentId || req || resp) with Agent's private key. Returns X-Reputation-Proof.
7
Client → On-chain new
Single "submit feedback" transaction. Internally:
  (a) FeedbackGateway.markUsed(hash) — dedup
  (b) ReputationRegistry.giveFeedback(agentId, 95, "x402") — on-chain reputation
  (c) If (b) reverts, FeedbackGateway.unmarkUsed(hash) — retriable
