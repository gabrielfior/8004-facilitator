⚙ Standard x402 (what the SDK gives us)

  • Payment flow — client gets 402, signs EIP-3009, retries, facilitator verifies + settles
  • PaymentMiddlewareASGI — intercepts requests with PAYMENT-SIGNATURE header, calls verify/settle
  • x402Client / x402Facilitator / x402ResourceServer — core classes for each party
  • ExactEvmServerScheme / ExactEvmFacilitatorScheme — EIP-3009 + Permit2 support
  • HTTP facilitator client — server calls facilitator's /verify and /settle via HTTP
  • PAYMENT-RESPONSE header — settlement info returned to client

✨ Our additions (ERC-8004 reputation layer)

  • ReputationMiddleware — computes reqHash = keccak256(reqBody), attaches to erc-8004 extension in 402
  • reqHash in 402 — agent advertises request commitment (tx hash unknown yet)
  • X-Reputation-Commitment header — convenience header with hash(agentId, reqHash, respHash)
  • FeedbackGateway.recordCommitment() — stores commitments[agentId][hash(agentId, reqHash, txHash)] after settlement
  • Settlement dedup — usedSettlements[txHash] enforces one settlement = one verified feedback
  • FeedbackGateway.submitFeedback() — verifies commitment + settlement dedup + hash dedup, delegates to ReputationRegistry
  • Direct on-chain feedback — client calls submitFeedback via EIP-7702 (no facilitator needed)
  • Agent registration — IdentityRegistry.register() mints agent NFT

Flow steps — standard vs new

1
Client → Resource Server x402
GET /weather. No payment header. Server returns 402 + X-Payment-Requirements (USDC $0.01 or DAI $0.01).
2
Client → Facilitator x402
Client signs token transfer (EIP-3009 for USDC, Permit2 for DAI). Sends POST /verify. Facilitator validates signature + simulates.
3
Client → Resource Server x402
Retries GET /weather with PAYMENT-SIGNATURE header. Middleware intercepts.
4
Server → Facilitator → On-chain x402
POST /settle. Facilitator executes transferWithAuthorization (USDC) or Permit2 Proxy settle (DAI).
5
Server → Client x402
200 OK + PAYMENT-RESPONSE header (settlement tx hash, payer).
6
ReputationMiddleware (on Server) new
Computes reqHash = keccak256(reqBody) on incoming request. Attaches to erc-8004 extension in 402 response.
7
Facilitator (or Agent) → On-chain new
After settlement: computes commitment = keccak256(agentId, reqHash, settlementTxHash). Calls FeedbackGateway.recordCommitment(agentId, commitment).
8
Client → On-chain new
Verifies commitments[agentId][hash(agentId, reqHash, txHash)] from chain. Calls submitFeedback(registry, params) via EIP-7702 — checks commitment + settlement dedup + hash dedup.
